     1‚Üí# CSS Patterns for Diagrams
     2‚Üí

    53‚Üí## Background Atmosphere
    54‚Üí
    55‚ÜíFlat backgrounds feel dead. Use subtle gradients or patterns.
    56‚Üí
    57‚Üí```css
    58‚Üí/* Radial glow behind focal area */
    59‚Üíbody {
    60‚Üí  background: var(--bg);
    61‚Üí  background-image: radial-gradient(ellipse at 50% 0%, var(--accent-dim) 0%, transparent 60%);
    62‚Üí}
    63‚Üí
    64‚Üí/* Faint dot grid */
    65‚Üíbody {
    66‚Üí  background-color: var(--bg);
    67‚Üí  background-image: radial-gradient(circle, var(--border) 1px, transparent 1px);
    68‚Üí  background-size: 24px 24px;
    69‚Üí}
    70‚Üí
    71‚Üí/* Diagonal subtle lines */
    72‚Üíbody {
    73‚Üí  background-color: var(--bg);
    74‚Üí  background-image: repeating-linear-gradient(
    75‚Üí    -45deg, transparent, transparent 40px,
    76‚Üí    var(--border) 40px, var(--border) 41px
    77‚Üí  );
    78‚Üí}
    79‚Üí
    80‚Üí/* Gradient mesh (pick 2-3 positioned radials) */
    81‚Üíbody {
    82‚Üí  background: var(--bg);
    83‚Üí  background-image:
    84‚Üí    radial-gradient(at 20% 20%, var(--node-a-dim) 0%, transparent 50%),

    88‚Üí
    89‚Üí## Link Styling
    90‚Üí
    91‚Üí**Never rely on browser default link colors.** The default blue (`#0000EE`) has poor contrast on dark backgrounds. Style links with `color: var(--accent)` and keep underlines for discoverability. On dark backgrounds, use bright accents (`#22d3ee`, `#34d399`, `#fbbf24`). On light backgrounds, use deeper tones (`#0891b2`, `#059669`, `#d97706`).
    92‚Üí
    93‚Üí## Section / Card Components
    94‚Üí
    95‚ÜíThe fundamental building block. A colored card representing a system component, pipeline step, or data entity.
    96‚Üí
    97‚Üí**IMPORTANT: Never use `.node` as a CSS class name.** Mermaid.js internally uses `.node` on its SVG `<g>` elements with `transform: translate(x, y)` for positioning. Any page-level `.node` styles (hover transforms, box-shadows, transitions) will leak into Mermaid diagrams and break their layout. Use `.ve-card` instead (namespaced to avoid collisions with CSS frameworks like Bootstrap/Tailwind that also use `.card`).
    98‚Üí
    99‚Üí```css
   100‚Üí.ve-card {
   101‚Üí  background: var(--surface);
   102‚Üí  border: 1px solid var(--border);
   103‚Üí  border-radius: 10px;
   104‚Üí  padding: 16px 20px;
   105‚Üí  position: relative;
   106‚Üí}
   107‚Üí
   108‚Üí/* Colored accent border (left or top) */
   109‚Üí.ve-card--accent-a {
   110‚Üí  border-left: 3px solid var(--node-a);
   111‚Üí}
   112‚Üí
   113‚Üí/* --- Depth tiers: vary card depth to signal importance --- */
   114‚Üí
   115‚Üí/* Elevated: KPIs, key sections, anything that should pop */
   116‚Üí.ve-card--elevated {
   117‚Üí  background: var(--surface-elevated);
   118‚Üí  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.08), 0 1px 2px rgba(0, 0, 0, 0.04);
   119‚Üí}
   120‚Üí
   121‚Üí/* Recessed: code blocks, secondary content, detail panels */
   122‚Üí.ve-card--recessed {
   123‚Üí  background: color-mix(in srgb, var(--bg) 70%, var(--surface) 30%);
   124‚Üí  box-shadow: inset 0 1px 3px rgba(0, 0, 0, 0.06);
   125‚Üí  border-color: var(--border);
   126‚Üí}
   127‚Üí
   128‚Üí/* Hero: executive summaries, focal elements ‚Äî demands attention */
   129‚Üí.ve-card--hero {
   130‚Üí  background: color-mix(in srgb, var(--surface) 92%, var(--accent) 8%);
   131‚Üí  box-shadow: 0 4px 20px rgba(0, 0, 0, 0.08), 0 1px 3px rgba(0, 0, 0, 0.04);
   132‚Üí  border-color: color-mix(in srgb, var(--border) 50%, var(--accent) 50%);
   133‚Üí}
   134‚Üí
   135‚Üí/* Glass: special-occasion overlay effect (use sparingly) */
   136‚Üí.ve-card--glass {
   137‚Üí  background: color-mix(in srgb, var(--surface) 60%, transparent 40%);
   138‚Üí  backdrop-filter: blur(12px);
   139‚Üí  -webkit-backdrop-filter: blur(12px);
   140‚Üí  border-color: rgba(255, 255, 255, 0.1);
   141‚Üí}
   142‚Üí
   143‚Üí/* Section label (monospace, uppercase, small) */
   144‚Üí.ve-card__label {
   145‚Üí  font-family: var(--font-mono);
   146‚Üí  font-size: 10px;
   147‚Üí  font-weight: 600;
   148‚Üí  text-transform: uppercase;
   149‚Üí  letter-spacing: 1.5px;
   150‚Üí  color: var(--node-a);
   151‚Üí  margin-bottom: 10px;
   152‚Üí  display: flex;
   153‚Üí  align-items: center;
   154‚Üí  gap: 8px;
   155‚Üí}
   156‚Üí
   157‚Üí/* Colored dot indicator */
   158‚Üí.ve-card__label::before {
   159‚Üí  content: '';
   160‚Üí  width: 8px;
   161‚Üí  height: 8px;
   162‚Üí  border-radius: 50%;
   163‚Üí  background: currentColor;
   164‚Üí}
   165‚Üí```

   167‚Üí## Code Blocks
   168‚Üí
   169‚ÜíCode blocks need explicit whitespace preservation and a max-height constraint. Without these, code runs together and long files overwhelm the page.
   170‚Üí
   171‚Üí### Basic Pattern
   172‚Üí
   173‚Üí```css
   174‚Üí.code-block {
   175‚Üí  font-family: var(--font-mono);
   176‚Üí  font-size: 13px;
   177‚Üí  line-height: 1.5;
   178‚Üí  background: var(--surface);
   179‚Üí  border: 1px solid var(--border);
   180‚Üí  border-radius: 8px;
   181‚Üí  padding: 16px;
   182‚Üí  overflow-x: auto;
   183‚Üí  /* CRITICAL: preserve line breaks and indentation */
   184‚Üí  white-space: pre-wrap;
   185‚Üí  word-break: break-word;
   186‚Üí}
   187‚Üí
   188‚Üí/* Constrain height for long code */
   189‚Üí.code-block--scroll {
   190‚Üí  max-height: 400px;
   191‚Üí  overflow-y: auto;
   192‚Üí}
   193‚Üí```
   194‚Üí
   195‚Üí```html
   196‚Üí<pre class="code-block code-block--scroll"><code>// Your code here
   197‚Üífunction example() {
   198‚Üí  return true;
   199‚Üí}</code></pre>
   200‚Üí```
   201‚Üí
   202‚Üí### With File Header
   203‚Üí
   204‚Üí```css
   205‚Üí.code-file {
   206‚Üí  border: 1px solid var(--border);
   207‚Üí  border-radius: 8px;
   208‚Üí  overflow: hidden;
   209‚Üí}
   210‚Üí
   211‚Üí.code-file__header {
   212‚Üí  display: flex;
   213‚Üí  align-items: center;
   214‚Üí  gap: 8px;
   215‚Üí  padding: 10px 16px;
   216‚Üí  background: var(--surface);
   217‚Üí  border-bottom: 1px solid var(--border);
   218‚Üí  font-family: var(--font-mono);
   219‚Üí  font-size: 12px;
   220‚Üí  color: var(--text-dim);
   221‚Üí}
   222‚Üí
   223‚Üí.code-file__body {
   224‚Üí  font-family: var(--font-mono);

   247‚Üí### Implementation Plans: Don't Dump Full Files
   248‚Üí
   249‚ÜíFor implementation plans and architecture docs, **don't display entire source files inline**. Instead:
   250‚Üí
   251‚Üí1. **Show structure, not code:**
   252‚Üí   ```html
   253‚Üí   <div class="file-structure">
   254‚Üí     <div class="file-structure__path">src/extension.ts</div>
   255‚Üí     <ul class="file-structure__outline">
   256‚Üí       <li><code>BOOMERANG_INSTRUCTIONS</code> ‚Äî System prompt for autonomous mode</li>
   257‚Üí       <li><code>clearState()</code> ‚Äî Reset extension state</li>
   258‚Üí       <li><code>updateStatus()</code> ‚Äî Update UI status indicator</li>
   259‚Üí       <li><code>/boomerang</code> command ‚Äî Start autonomous task</li>
   260‚Üí       <li><code>/boomerang-cancel</code> command ‚Äî Cancel active task</li>
   261‚Üí       <li><code>before_agent_start</code> hook ‚Äî Inject instructions</li>
   262‚Üí       <li><code>agent_end</code> hook ‚Äî Generate summary</li>
   263‚Üí     </ul>
   264‚Üí   </div>
   265‚Üí   ```
   266‚Üí
   267‚Üí2. **Use collapsible sections for full code:**
   268‚Üí   ```html
   269‚Üí   <details class="collapsible">
   270‚Üí     <summary>Full implementation (87 lines)</summary>
   271‚Üí     <pre class="code-file__body"><code>...</code></pre>
   272‚Üí   </details>
   273‚Üí   ```
   274‚Üí
   275‚Üí3. **Show key snippets only:**
   276‚Üí   ```html
   277‚Üí   <p>The core logic intercepts task completion:</p>
   278‚Üí   <pre class="code-block"><code>pi.on("agent_end", async () => {
   279‚Üí     const summary = generateSummary(workEntries);
   280‚Üí     boomerangComplete = true;
   281‚Üí   });</code></pre>
   282‚Üí   ```
   283‚Üí
   284‚Üí**Anti-patterns:**
   285‚Üí- Displaying full source files inline (100+ lines overwhelming the page)
   286‚Üí- Code blocks without `white-space: pre-wrap` (code runs together into unreadable wall)
   287‚Üí- No height constraint on long code (page becomes endless scroll)
   288‚Üí
   289‚ÜíIf someone needs the full file, put it in a collapsible section or link to it.
   290‚Üí
   291‚Üí## Overflow Protection
   292‚Üí
   293‚ÜíGrid and flex children default to `min-width: auto`, which prevents them from shrinking below their content width. Long text, inline code badges, and non-wrapping elements will blow out containers.
   294‚Üí
   295‚Üí### Global rules
   296‚Üí
   297‚Üí```css
   298‚Üí/* Every grid/flex child must be able to shrink */
   299‚Üí.grid > *, .flex > *,
   300‚Üí[style*="display: grid"] > *,
   301‚Üí[style*="display: flex"] > * {
   302‚Üí  min-width: 0;
   303‚Üí}
   304‚Üí
   305‚Üí/* Long text wraps instead of overflowing */
   306‚Üíbody {
   307‚Üí  overflow-wrap: break-word;
   308‚Üí}
   309‚Üí```
   310‚Üí
   311‚Üí### Side-by-side comparison panels
   312‚Üí
   313‚Üí```css
   314‚Üí.comparison {
   315‚Üí  display: grid;
   316‚Üí  grid-template-columns: 1fr 1fr;
   317‚Üí  gap: 16px;
   318‚Üí}
   319‚Üí
   320‚Üí.comparison > * {
   321‚Üí  min-width: 0;
   322‚Üí  overflow-wrap: break-word;
   323‚Üí}
   324‚Üí
   325‚Üí@media (max-width: 768px) {
   326‚Üí  .comparison { grid-template-columns: 1fr; }
   327‚Üí}
   328‚Üí```
   329‚Üí
   330‚Üí### Never use `display: flex` on `<li>` for marker characters
   331‚Üí
   332‚ÜíUsing `display: flex` on a list item to position a `::before` marker creates an anonymous flex item for the remaining text content. That anonymous flex item gets `min-width: auto` and you **cannot** set `min-width: 0` on anonymous boxes. Lines with many inline `<code>` badges will overflow their container with no CSS fix possible.
   333‚Üí
   334‚ÜíUse absolute positioning for markers instead:
   335‚Üí
   336‚Üí```css
   337‚Üí/* WRONG ‚Äî causes overflow with inline code badges */
   338‚Üíli {
   339‚Üí  display: flex;
   340‚Üí  align-items: baseline;
   341‚Üí  gap: 6px;
   342‚Üí}
   343‚Üíli::before {
   344‚Üí  content: '‚Ä∫';
   345‚Üí  flex-shrink: 0;
   346‚Üí}
   347‚Üí
   348‚Üí/* RIGHT ‚Äî text wraps normally */
   349‚Üíli {
   350‚Üí  padding-left: 14px;
   351‚Üí  position: relative;
   352‚Üí}
   353‚Üíli::before {
   354‚Üí  content: '‚Ä∫';
   355‚Üí  position: absolute;
   356‚Üí  left: 0;
   357‚Üí}
   358‚Üí```
   359‚Üí
   360‚Üí### List markers overlapping container borders
   361‚Üí
   362‚ÜíBy default, `list-style-position: outside` places list markers (bullets, numbers) outside the content box. When lists are inside bordered containers (cards, callout boxes), the markers can overlap or extend beyond the border.
   363‚Üí
   364‚Üí```css
   365‚Üí/* WRONG ‚Äî markers overlap container border */
   366‚Üí.card ol, .card ul {
   367‚Üí  padding-left: 20px;  /* Not enough for outside markers */
   368‚Üí}
   369‚Üí
   370‚Üí/* RIGHT ‚Äî use inside positioning */
   371‚Üí.card ol, .card ul {
   372‚Üí  list-style-position: inside;
   373‚Üí}
   374‚Üí
   375‚Üí/* OR ‚Äî adequate padding for outside markers */
   376‚Üí.card ol, .card ul {
   377‚Üí  padding-left: 2em;  /* ~32px gives room for markers */
   378‚Üí}
   379‚Üí
   380‚Üí/* OR ‚Äî custom markers with absolute positioning (most control) */
   381‚Üí.card ol {
   382‚Üí  list-style: none;
   383‚Üí  padding-left: 0;
   384‚Üí  counter-reset: item;
   385‚Üí}
   386‚Üí.card ol li {
   387‚Üí  counter-increment: item;
   388‚Üí  padding-left: 2em;
   389‚Üí  position: relative;
   390‚Üí}
   391‚Üí.card ol li::before {
   392‚Üí  content: counter(item) ".";
   393‚Üí  position: absolute;
   394‚Üí  left: 0;
   395‚Üí  color: var(--accent);
   396‚Üí  font-weight: 600;
   397‚Üí}
   398‚Üí```
   399‚Üí
   400‚Üí**Rule of thumb:** Any `<ol>` or `<ul>` inside a bordered container needs either `list-style-position: inside` or `padding-left: 2em` minimum. The default 20px padding is not enough for outside-positioned markers.

   402‚Üí## Mermaid Containers
   403‚Üí
   404‚ÜíMermaid diagrams have two common layout issues: they render too small to read, and they left-align in their container leaving awkward dead space (especially for narrow vertical flowcharts).
   405‚Üí
   406‚Üí### Centering (Required)
   407‚Üí
   408‚ÜíMermaid SVGs render at a fixed size based on content. Without explicit centering, they default to top-left alignment. **Always center Mermaid diagrams** ‚Äî narrow vertical flowcharts look particularly bad when left-aligned in a wide container.
   409‚Üí
   410‚Üí```css
   411‚Üí/* WRONG ‚Äî diagram hugs left edge */
   412‚Üí.mermaid-container {
   413‚Üí  padding: 24px;
   414‚Üí  border: 1px solid var(--border);
   415‚Üí}
   416‚Üí
   417‚Üí/* RIGHT ‚Äî diagram centers in container */
   418‚Üí.mermaid-wrap {
   419‚Üí  display: flex;
   420‚Üí  justify-content: center;
   421‚Üí  align-items: flex-start;  /* or center for shorter diagrams */
   422‚Üí  padding: 24px;
   423‚Üí  border: 1px solid var(--border);
   424‚Üí}
   425‚Üí```
   426‚Üí
   427‚Üí### Scaling Small Diagrams
   428‚Üí
   429‚ÜíMermaid sizes diagrams based on content, not container. Complex diagrams with many nodes render small to fit everything, leaving the text nearly unreadable. Three fixes:
   430‚Üí
   431‚Üí**1. Increase fontSize in themeVariables** (most effective):
   432‚Üí```javascript
   433‚Üímermaid.initialize({
   434‚Üí  theme: 'base',
   435‚Üí  themeVariables: {
   436‚Üí    fontSize: '18px',  // default is 16px, bump to 18-20px for complex diagrams
   437‚Üí  }
   438‚Üí});
   439‚Üí```
   440‚Üí
   441‚Üí**2. CSS zoom** for diagrams that still render too small:
   442‚Üí```css
   443‚Üí.mermaid-wrap--scaled .mermaid {
   444‚Üí  zoom: 1.3;
   445‚Üí}
   446‚Üí```
   447‚Üí
   448‚Üí**3. Constrain container width** so the diagram doesn't float in dead space:
   449‚Üí```css
   450‚Üí.mermaid-wrap--constrained {
   451‚Üí  max-width: 800px;
   452‚Üí  margin: 0 auto;
   453‚Üí}
   454‚Üí```
   455‚Üí
   456‚Üí**Rule of thumb:** If the diagram has 10+ nodes or the text is smaller than 12px rendered, increase fontSize to 18-20px or apply CSS zoom.
   457‚Üí
   458‚Üí### Zoom Controls
   459‚Üí
   460‚ÜíAdd zoom controls to every `.mermaid-wrap` container for complex diagrams.
   461‚Üí
   462‚Üí**Small diagrams in slides.** If a diagram has fewer than ~7 nodes with no branching, it will render tiny in a full-viewport slide container. For simple linear flows (A ‚Üí B ‚Üí C ‚Üí D), use CSS pipeline cards instead of Mermaid ‚Äî see `slide-patterns.md` "CSS Pipeline Slide." Reserve Mermaid for complex graphs where automatic edge routing is actually needed.
   463‚Üí
   464‚Üí### Full Pattern
   465‚Üí
   466‚Üí```css
   467‚Üí.mermaid-wrap {
   468‚Üí  position: relative;

   543‚Üí
   544‚Üí**Why zoom instead of transform?**
   545‚Üí
   546‚ÜíCSS `transform: scale()` only changes visual appearance ‚Äî the element's layout box stays the same size. When you scale from `center center`, content expands upward and leftward into negative coordinate space. Scroll containers can't scroll to negative positions, so the top and left of the zoomed content get clipped.
   547‚Üí
   548‚ÜíCSS `zoom` actually changes the element's layout size. The content grows downward and rightward like any other growing element, staying fully scrollable.
   549‚Üí

   566‚Üí### JavaScript
   567‚Üí
   568‚ÜíAdd once at the end of the page. Handles button clicks and scroll-to-zoom on all `.mermaid-wrap` containers:
   569‚Üí
   570‚Üí```javascript
   571‚Üí// Match this to the CSS zoom value (or 1 if not set)
   572‚Üívar INITIAL_ZOOM = 1.4;
   573‚Üí
   574‚Üífunction zoomDiagram(btn, factor) {
   575‚Üí  var wrap = btn.closest('.mermaid-wrap');
   576‚Üí  var target = wrap.querySelector('.mermaid');
   577‚Üí  var current = parseFloat(target.dataset.zoom || INITIAL_ZOOM);
   578‚Üí  var next = Math.min(Math.max(current * factor, 0.5), 5);
   579‚Üí  target.dataset.zoom = next;
   580‚Üí  target.style.zoom = next;
   581‚Üí}
   582‚Üí
   583‚Üífunction resetZoom(btn) {
   584‚Üí  var wrap = btn.closest('.mermaid-wrap');
   585‚Üí  var target = wrap.querySelector('.mermaid');
   586‚Üí  target.dataset.zoom = INITIAL_ZOOM;
   587‚Üí  target.style.zoom = INITIAL_ZOOM;
   588‚Üí}
   589‚Üí
   590‚Üídocument.querySelectorAll('.mermaid-wrap').forEach(function(wrap) {
   591‚Üí  // Ctrl/Cmd + scroll to zoom
   592‚Üí  wrap.addEventListener('wheel', function(e) {
   593‚Üí    if (!e.ctrlKey && !e.metaKey) return;
   594‚Üí    e.preventDefault();
   595‚Üí    var target = wrap.querySelector('.mermaid');
   596‚Üí    var current = parseFloat(target.dataset.zoom || INITIAL_ZOOM);
   597‚Üí    var factor = e.deltaY < 0 ? 1.1 : 0.9;
   598‚Üí    var next = Math.min(Math.max(current * factor, 0.5), 5);
   599‚Üí    target.dataset.zoom = next;
   600‚Üí    target.style.zoom = next;
   601‚Üí  }, { passive: false });
   602‚Üí
   603‚Üí  // Click-and-drag to pan
   604‚Üí  var startX, startY, scrollL, scrollT;
   605‚Üí  wrap.addEventListener('mousedown', function(e) {
   606‚Üí    if (e.target.closest('.zoom-controls')) return;
   607‚Üí    wrap.classList.add('is-panning');
   608‚Üí    startX = e.clientX;
   609‚Üí    startY = e.clientY;
   610‚Üí    scrollL = wrap.scrollLeft;
   611‚Üí    scrollT = wrap.scrollTop;
   612‚Üí  });
   613‚Üí  window.addEventListener('mousemove', function(e) {
   614‚Üí    if (!wrap.classList.contains('is-panning')) return;
   615‚Üí    wrap.scrollLeft = scrollL - (e.clientX - startX);
   616‚Üí    wrap.scrollTop = scrollT - (e.clientY - startY);
   617‚Üí  });
   618‚Üí  window.addEventListener('mouseup', function() {
   619‚Üí    wrap.classList.remove('is-panning');
   620‚Üí  });
   621‚Üí});
   622‚Üí```
   623‚Üí
   624‚ÜíScroll-to-zoom requires Ctrl/Cmd+scroll to avoid hijacking normal page scroll. Cursor changes to `grab`/`grabbing` to signal pan mode. The zoom range is capped at 0.5x‚Äì5x.

   935‚Üí## Animations
   936‚Üí
   937‚Üí### Staggered Fade-In on Load
   938‚Üí
   939‚ÜíDefine the keyframe once, then stagger via a `--i` CSS variable set per element. This approach works regardless of DOM nesting or interleaved non-animated elements (unlike `nth-child` which breaks when siblings aren't all the same type).
   940‚Üí
   941‚Üí```css
   942‚Üí@keyframes fadeUp {
   943‚Üí  from { opacity: 0; transform: translateY(12px); }
   944‚Üí  to { opacity: 1; transform: translateY(0); }
   945‚Üí}
   946‚Üí
   947‚Üí.ve-card {
   948‚Üí  animation: fadeUp 0.4s ease-out both;
   949‚Üí  animation-delay: calc(var(--i, 0) * 0.05s);
   950‚Üí}
   951‚Üí```

  1005‚Üí### CSS Counter (for hero numbers without JS)
  1006‚Üí
  1007‚ÜíUses `@property` to animate a custom property as an integer, then display it via `counter()`. No JS required. Falls back to showing the final value immediately in browsers without `@property` support.
  1008‚Üí
  1009‚Üí```css
  1010‚Üí@property --count {
  1011‚Üí  syntax: '<integer>';
  1012‚Üí  initial-value: 0;
  1013‚Üí  inherits: false;
  1014‚Üí}
  1015‚Üí
  1016‚Üí@keyframes countUp {
  1017‚Üí  to { --count: var(--target); }
  1018‚Üí}
  1019‚Üí
  1020‚Üí.kpi-card__value--animated {
  1021‚Üí  --target: 247;
  1022‚Üí  counter-reset: val var(--count);
  1023‚Üí  animation: countUp 1.2s ease-out forwards;
  1024‚Üí}
  1025‚Üí
  1026‚Üí.kpi-card__value--animated::after {
  1027‚Üí  content: counter(val);
  1028‚Üí}
  1029‚Üí```
  1030‚Üí
  1031‚Üí### Choreography
  1032‚Üí
  1033‚ÜíDon't use the same animation for everything. Mix types by element role, with easing stagger (fast-then-slow, not linear):
  1034‚Üí
  1035‚Üí- **Cards**: `fadeUp` ‚Äî the default entrance, reliable and subtle
  1036‚Üí- **KPI / badges**: `fadeScale` ‚Äî scale draws the eye to important numbers
  1037‚Üí- **SVG connectors**: `drawIn` ‚Äî reveals flow direction, pairs with card stagger
  1038‚Üí- **Hero numbers**: `countUp` ‚Äî counting motion signals "this number matters"
  1039‚Üí- **Stagger timing**: `calc(var(--i) * 0.06s)` with lower `--i` values on important elements so they appear first
  1040‚Üí
  1041‚Üí### Respect Reduced Motion
  1042‚Üí```css
  1043‚Üí@media (prefers-reduced-motion: reduce) {
  1044‚Üí  *, *::before, *::after {
  1045‚Üí    animation-duration: 0.01ms !important;
  1046‚Üí    animation-iteration-count: 1 !important;
  1047‚Üí    transition-duration: 0.01ms !important;
  1048‚Üí  }
  1049‚Üí}
  1050‚Üí```

  1620‚Üí
  1621‚ÜíAvoid these in reading-first content:
